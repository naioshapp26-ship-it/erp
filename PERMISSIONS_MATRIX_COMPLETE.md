## 🗄️ بنية قاعدة البيانات

### الجداول الرئيسية

#### 1. `permission_levels`
```sql
- id: SERIAL PRIMARY KEY
- level_code: VARCHAR(50) UNIQUE
- level_name_ar: VARCHAR(100)
- level_name_en: VARCHAR(100)
- description_ar: TEXT
- description_en: TEXT
- color_code: VARCHAR(20)
- priority_order: INTEGER
```

#### 2. `systems`
```sql
- id: SERIAL PRIMARY KEY
- system_code: VARCHAR(50) UNIQUE
- system_name_ar: VARCHAR(200)
- system_name_en: VARCHAR(200)
- description_ar: TEXT
- main_functions_ar: TEXT[]
- is_active: BOOLEAN
- display_order: INTEGER
```

#### 3. `roles` (محدّث)
```sql
- id: SERIAL PRIMARY KEY
- name: VARCHAR(50) UNIQUE
- name_ar: VARCHAR(100)
- description: TEXT
- level: VARCHAR(20)
- hierarchy_level: INTEGER           -- جديد
- job_title_ar: VARCHAR(200)         -- جديد
- job_title_en: VARCHAR(200)         -- جديد
- min_approval_limit: DECIMAL(15,2)  -- جديد
- max_approval_limit: DECIMAL(15,2)  -- جديد (NULL = غير محدود)
- approval_notes_ar: TEXT            -- جديد
- is_active: BOOLEAN
```

#### 4. `role_system_permissions` (مصفوفة الصلاحيات)
```sql
- id: SERIAL PRIMARY KEY
- role_id: INTEGER -> roles(id)
- system_id: INTEGER -> systems(id)
- permission_level_id: INTEGER -> permission_levels(id)
- is_active: BOOLEAN
- notes: TEXT
- UNIQUE(role_id, system_id)
```

#### 5. `security_policies`
```sql
- id: SERIAL PRIMARY KEY
- policy_code: VARCHAR(50) UNIQUE
- policy_name_ar: VARCHAR(200)
- policy_name_en: VARCHAR(200)
- description_ar: TEXT
- responsible_role: VARCHAR(100)
- enforcement_type: VARCHAR(50)
- is_active: BOOLEAN
```

## 🚀 كيفية الاستخدام

### 1. التحقق من صلاحية مستخدم لنظام معين
```sql
-- مثال: هل المستخدم #5 لديه صلاحية VIEW على نظام FINANCE؟
SELECT 
    u.username,
    s.system_name_ar,
    pl.level_name_ar
FROM users u
JOIN user_roles ur ON u.id = ur.user_id
JOIN roles r ON ur.role_id = r.id
JOIN role_system_permissions rsp ON r.id = rsp.role_id
JOIN systems s ON rsp.system_id = s.id
JOIN permission_levels pl ON rsp.permission_level_id = pl.id
WHERE u.id = 5 
AND s.system_code = 'FINANCE'
AND pl.priority_order <= (SELECT priority_order FROM permission_levels WHERE level_code = 'VIEW');
```

### 2. التحقق من حد الموافقة المالية
```sql
-- مثال: هل المستخدم #5 يمكنه الموافقة على 100,000 ريال؟
SELECT 
    u.username,
    r.job_title_ar,
    r.min_approval_limit,
    r.max_approval_limit,
    CASE 
        WHEN r.max_approval_limit IS NULL THEN 'نعم - صلاحيات غير محدودة'
        WHEN r.max_approval_limit >= 100000 THEN 'نعم'
        ELSE 'لا'
    END AS can_approve_100k
FROM users u
JOIN user_roles ur ON u.id = ur.user_id
JOIN roles r ON ur.role_id = r.id
WHERE u.id = 5;
```

### 3. عرض صلاحيات مستخدم كاملة
```sql
SELECT 
    r.job_title_ar AS الوظيفة,
    s.system_name_ar AS النظام,
    pl.level_name_ar AS المستوى,
    r.max_approval_limit AS حد_الموافقة
FROM users u
JOIN user_roles ur ON u.id = ur.user_id
JOIN roles r ON ur.role_id = r.id
JOIN role_system_permissions rsp ON r.id = rsp.role_id
JOIN systems s ON rsp.system_id = s.id
JOIN permission_levels pl ON rsp.permission_level_id = pl.id
WHERE u.id = 5
AND rsp.is_active = TRUE
ORDER BY s.display_order, pl.priority_order;
```
